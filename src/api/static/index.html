Getting started

        
    
          
  1. Start the dashboard on your PC: .\scripts\run_dashboard.ps1
    2. 
          
  2. Open http://localhost:8000 on this machine, or http://<your-pc-ip>:8000 from a phone on the same Wi‑Fi.
    3. 
          
  3. Set DASHBOARD_API_KEY in your .env file, then save it in the Control API key field above (needed for Start/Stop and backtests).
    4. 
          
  4. Use Overview → Start to launch paper trading, or run a backtest first to validate the strategy.
    5. 
        

        
Do not browse to 0.0.0.0 — that is the server bind address, not a user URL. If localhost shows an error, restart via the PowerShell script (it clears stale processes on port 8000).

      


      

        
Control API key

        
Protects write actions on your home network: starting/stopping the bot and launching backtests. Read-only tabs (Overview, Positions, etc.) work without the key.

        
    
          
  • Generate any long random string and add it to .env as DASHBOARD_API_KEY=...
    • 
          
  • Paste the same value here and click Save key — stored only in this browser’s local storage
    • 
          
  • Restart the dashboard server after changing .env
    • 
        

      


      

        

          
Overview

          
Live snapshot of your Alpaca paper account and bot status. Refreshes every few seconds automatically.

          
    
            
  • Equity — total account value
    • 
            
  • Daily P&L — change vs. prior close
    • 
            
  • Positions — count of open holdings
    • 
            
  • Market — US market hours (bot may still run in paper mode when closed)
    • 
            
  • Start / Stop — runs src/main.py --mode paper as a background process; requires API key
    • 
            
  • Last heartbeat — written by the bot each trading cycle; if missing while “running”, the process may have crashed
    • 
          

        


        

          
P&L

          
Historical daily profit and loss from Alpaca portfolio history — end-of-day equity vs. the prior trading day.

          
    
            
  • 1M / 3M / 6M / 1Y / All — time range for the chart and table
    • 
            
  • Period P&L — sum of daily changes in the selected window
    • 
            
  • Win / loss days — count of up vs. down days
    • 
          

        


        

          
Positions

          
Open positions from your Alpaca paper account, synced on each refresh.

          
    
            
  • Qty — shares held (negative = short)
    • 
            
  • Avg / Current — average entry vs. latest price
    • 
            
  • Unrealized — open P&L on the position
    • 
          

          
Data comes directly from Alpaca, not from the bot process — you can view positions even when the bot is stopped.

        


        

          
Backtest

          
Run historical simulations using saved data and your trained XGBoost model. Results are stored under cache/dashboard/backtests/.

          
    
            
  • Symbols — space-separated tickers (e.g. AAPL MSFT)
    • 
            
  • Dates — use ranges covered by your data (parquet is roughly Jun–Nov 2024 today)
    • 
            
  • Capital — starting cash for the simulation
    • 
            
  • Return / Sharpe / Max DD / Trades — summary metrics when complete
    • 
            
  • Equity chart — portfolio value over the backtest period
    • 
          

          
First run can take several minutes (model + FinBERT load). Requires API key. Progress polls automatically until done.

        


        

          
Models

          
Lists trained models found in the models/ folder (newest version per model name).

          
    
            
  • MarketRegimeXGBoost — primary model from train_simple_massive.py
    • 
            
  • Metrics — validation accuracy and training metadata from metadata.json
    • 
          

          
Train with: python training/train_simple_massive.py --data data/historical_data_quality.parquet --symbols 10

        


        

          
Activity

          
Recent orders from your Alpaca paper account (buys, sells, fills, cancellations). Useful to confirm the bot is placing trades.

          
Shows the last 10 orders. Status values include filled, new, canceled, etc.

        


        

          
Assistant

          
Chat with a local Ollama model that receives a live snapshot of your paper account, bot heartbeat, models, and recent backtests — so you can ask things like “why no trades today?” or “explain my largest position.”

          
    
            
  • Setup — Install Ollama, run ollama pull llama3.2:1b (default; switch models in the Assistant tab)
    • 
            
  • Model picker — dropdown lists all models Ollama has pulled on this PC; choice is saved in your browser
    • 
            
  • OLLAMA_BASE_URL — default http://127.0.0.1:11434 (must run on the PC hosting the dashboard)
    • 
            
  • Tools — the model can refresh Alpaca snapshot and bot status mid-conversation (same read-only data as the other tabs)
    • 
            
  • Alpaca MCP in Cursor — for IDE-side questions, add the official Alpaca MCP server in Cursor settings (see below)
    • 
          

        

      


      

        
Alpaca MCP in Cursor (optional)

        
The dashboard assistant uses your existing .env Alpaca keys via the API. For richer tooling inside Cursor, install the official Alpaca MCP server and point it at the same keys:

        
{
  "mcpServers": {
    "alpaca": {
      "command": "uvx",
      "args": ["alpaca-mcp-server", "serve"],
      "env": {
        "ALPACA_API_KEY": "your_key",
        "ALPACA_SECRET_KEY": "your_secret",
        "ALPACA_PAPER": "true"
      }
    }
  }
}

        
Use paper keys from Alpaca dashboard. Package: pip install alpaca-mcp-server or uvx alpaca-mcp-server. Docs: github.com/alpacahq/alpaca-mcp-server

      


      

        
LAN access from phone or tablet

        
    
          
  • If tabs or data look missing after an update, force-refresh: Safari → reload icon (or close tab and reopen). iPad caches scripts aggressively.
    • 
          
  • PC and device must be on the same Wi‑Fi
    • 
          
  • Find your PC IP: ipconfig → IPv4 Address (e.g. 192.168.0.136)
    • 
          
  • Open http://<that-ip>:8000
    • 
          
  • If it fails, allow inbound TCP port 8000 in Windows Firewall (Private network only)
    • 
        

      


      

        
Typical workflow

        

          
1. Validate — Run a backtest on dates you have data for; check return, drawdown, and trade count.

          
2. Paper trade — Start the bot from Overview; watch Activity for orders and Positions for holdings.

          
3. Monitor — Keep the dashboard open on a phone via LAN, or hit Refresh periodically.

          
4. Stop — Use Stop when you want to halt new signals (existing positions remain at Alpaca until closed).

        

      


      

        
Important notes

        
    
          
  • This is paper trading only unless you change bot mode manually — no real money.
    • 
          
  • The dashboard and bot are separate processes; stopping the dashboard does not stop the bot.
    • 
          
  • Backtest performance does not guarantee live results — slippage, sentiment, and data gaps affect real runs.
    • 
          
  • API docs for developers: /docs on this server.
